# Transformer

最初的的 2 个主要部分组成：编码器（encoder）和解码器（decoder）。

编码器：对整个输入文本进行预处理，提取相关上下文信息
解码器：利用编码器提供的上下文信息来生成文本

## Transformer 大语言模型如何工作

### 词袋模型（Bag of Words）

是一种用**大型稀疏向量**或**数字数组**来表示单词的算法,只关心一个词有没有出现，出现几次，不理解词义。

标记化（Tokenization）：把人类输入的文本，切分成模型能处理的最小单位 token。

token: 模型能理解的最小单位，通常是单词或子词。

词表（vocabulary）：模型 tokenizer 认识的所有 token 集合，记录了 token 和对应的数字 ID，tokenizer 会用它来查 ID

向量表示法（Vector Representation）：把 token 转换为数字向量，便于模型处理。

### Word2Vec（word embeddings 的一种方法）

词袋法的缺陷：认为语言是一种完全被词汇直接描述的工具，忽略了语义和含义。

Word2Vec：把单词变成带有语义关系的数字向量。

本质运用了神经网络技术，通过大量语料库训练，让模型学习到单词之间的语义关系。

![Word2Vec](https://cdn.heritcoin.com/sky/official/d/image/20260615/c170547do80da7vcat-W2054H634.png)

- **word embedding**：将单词转换为向量表示的方法。模型会为每个单词/token 分配一组固定长度的数值，这些数值通常先随机初始化，再通过训练逐步学习出语义信息。

![嵌入](https://cdn.heritcoin.com/sky/official/d/image/20260615/c17154192j74lrk4ig-W1750H836.png)
**嵌入（Embedding）**：由一组高维度数字向量组成，这些数字向量由模型通过训练学习得出，通常介于-1 ～ 1 之间，一个模型里，一个嵌入所拥有的属性或值的数量被称为维度（dimension）。通常是固定的。比如：`cat  → [0.12, -0.45, 0.08, 0.91, -0.33]`

tokenizers（分词器）：具有固定的词汇表，不能代表所有存在的单词。还负责把 token 映射成模型认识的数字。输出的不仅是 id，还有一些辅助信息，比如：token 的类型、token 的子词、token 的词性等。

### 编码和解码（Encoding and Decoding）

Word2Vec：创建的是静态嵌入（static embedding），无论上下文如何都生成相同的嵌入向量。

通过 Recurrent Neural Network（RNN）：适合处理序列数据的神经网络，根据上下文动态生成嵌入向量。不是孤立看每个词，而是会把前面处理过的信息传递到后面，用来理解整个句子的上下文。

RNN 被用于两个任务：编码或表示数据。

**Autoregressive Model（自回归模型）**：在生成文本时，根据前面生成的文本预测下一个词。

#### encoding and decoding context

比如我们输入一个句子，可以使用 Word2Vec 构建输出向量，
编码器能够一次性处理整个序列并考虑了嵌入信息的上下文情况。编码的目标是尽可能准确地表示输出内容，并以嵌入形式（embedding）生成上下文信息。
解码器则通过利用之前生成的上下文嵌入信息来实现（embedding）负责生成语言数据，最终生成所需结果。
这些输出的 token 是一次一个生成的，这种类型叫做**Autoregressive（自回归模型）**。

痛点：这种上下文嵌入方式（embedding）在处理长文本时，效果不佳，因为常见的 embedding 模型会把一段文本压缩成一个固定长度向量。长文本信息太多，被压缩后就容易丢细节。而且很多 embedding 模型都有最大输入长度。

因此有了注意力机制（Attention Mechanism），注意力机制能够帮助模型在处理长文本时，更好地关注重要信息，从而提高生成质量。
有了注意力机制之后，编码器会把每个输入 token 的隐藏状态都提供给解码器，解码器在每一步生成时动态选择应该关注哪些输入位置。
解码器利用注意力机制来关注整个序列，从而产生了语言。

> 没有注意力机制时：只传一个上下文向量。
> 加入 Attention 后，编码器不会只给解码器一个最终向量，而是把每个输入词对应的隐藏状态都保留下来。

但是这种 Encoder-Decoder + RNN + Attention 架构有一个问题，就是编码器和解码器之间是单向的，不能同时处理输入和输出。也就是无法实现并行处理。

## 如何利用注意力机制

### transformer works

由多个编码器和解码器模块堆叠而成，都采用了之前提到的注意力机制。
通过堆叠这些模块，可以增强编码器和解码器的处理能力。

![transformer works](https://cdn.heritcoin.com/sky/official/d/image/20260616/c163603i1ebgnnhnne-W2136H880.png)

transformer encoder 编码器：输入会被转换成嵌入格式（embedding）,但是不使用 Word2Vec 这种静态嵌入，而是随机值开始进行建模；然后通过注意力机制，让模型关注到输入中最重要的信息，并生成嵌入模型，包含了更多上下文相关的信息。然后被传递给前馈神经网络进行处理，前馈神经网络会对嵌入进行进一步的转换，最终生成输出。

注意力机制：不需要处理 2 个独立的输入数据，在序列中只处理一个序列，通过将其与自身进行比较，得到输入值。

解码器：可以处理任何输入数据，先生成一些单词，然后将其传递给掩码自注意力机制。与编码器类似，也需经过处理，生成中间层嵌入，并传递。同时还包含了编码器中的嵌入信息，一起发送给另一个注意力网络。就能处理到已经生成的内容，以及目前已有的内容。输出会被传递给神经网络进行处理，最后生成序列中的下一个单词。

### Bidirectional Encoder Representations from Transformers（双向编码,BERT）

是一种仅基于编码器的架构，专注于解决特定问题，在表示语言和生成上下文词嵌入方面有重要作用。
这些编码器模块和之前看到的是一样的，在神经网络中，自我注意力有重要作用。输入中包含一个额外的标记或关键词，CLS 或分类令牌，它被用做整个输入的表示方式。通常将这种令牌用做精细调度的输入嵌入对模型进行调优，让他在特定任务上表现更好。

可以使用一种叫做**掩码语言建模**的技术训练出 BERT 模型。掩码语言建模：在训练时，随机遮盖一些单词，让模型预测被遮盖的单词。
通常采用两步法进行：

1. 第一步需要对数据应用掩码语言建模技术，大量的数据，这被称为**预训练**。
2. 第二步：对模型进行微调

### 生成式模型（Generative Models）

采用和上述不同的方式，生成模型通常只包含解码器层。
输入数据,随机初始化嵌入向量,输入数据被直接传递给解码器。解码器再次使用掩码技术。然后这种自注意力会被传递给神经网络。最后生成下一个单词。

### 上下文长度

从一个输入序列开始，加上之前已有的信息，生成的 tokens 代表了当前的上下文长度。

## Tokenizers

比如有一串文本："Have the bards who。。。“
系统会先将文本拆分成若干较小片段(tokens)：“have”, "the", "bards", "who"，这种事通过 tokenization 来实现的。（叫做分词处理）
每个 token 会去词表 vocabulary 里查对应的 ID，随后都会被转换为一个数字形式表示法，也称为嵌入方法（embedding），这些都是向量值。
这些嵌入结构是静态的，每个嵌入都是固定的。是独立于所有其他嵌入和 token 而创建的
这些嵌入数据由大语言模型进行处理
然后被转换为上下文相关的嵌入形式，这些基于上下文的嵌入方式，对于每个输入 token 来说都是唯一的。
这些嵌入结构可以是一个模型的输出结果，也会使用这些数据进行输出的生成,在生成式模型的情况下，这可以是另一个标记

tokenization 工作原理：给定一个输入文本，tokenization 会将其拆分成若干较小片段(tokens)，这些 tokens 可以是一个单词，一个字符，一个符号，一个子词等。每个 token 都对应有一个固定值 ID，可以用于编码和解码这些 token。
这些输入将被传递给语言模型进行处理，内部操作实际上就产生了 token embeddings，生成模型的输出将是一个其他的 token ID,然后解码后就表示一个实际的 token。
