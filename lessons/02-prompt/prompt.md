# 提示词

## 2 种 LLMs 模型

1. 基础 LLM：根据文本训练数据来预测下一个词
2. 指令调整 LLM：被用于 LLM 研究和实践，训练来遵循指令的。
   2-1 训练方式：使用在大量文本数据上训练的基础 LLM，然后使用指令调进一步训练和微调。通常会使用一种 RLHF（Reinforcement Learning from Human Feedback:人类反馈强化学习）的方式来训练。

## 原则

### 原则 1: 编写清晰和具体的说明

#### 4 个策略

- 使用界定符清楚地指示输入的不同部分（html 标签、引号、分隔符等）
- 要求结构化地输出
- 检查条件是否满足（是否满足某些条件才输出）
- 少量事例提示

### 给模型一些时间来思考

可以尝试重新构思查询，要求模型在提供最终答案之前进行相关推理

#### 策略

- 明确指定完成任务所需的步骤
- 在匆忙得出结论之前让模型自己解决问题

### 局限性

幻觉：模型在回答时，可能会出现不准确或不真实的信息。上述方案可以尽量减少幻觉。
额外方法：可以先让模型在上下文中找到相关的引用，然后根据引用回答问题。

## 迭代式提示词开发过程

其实就是根据结果不断调整提示词，直到得到满意的结果。

## 对文本进行摘要

根据文本目标用途，进行侧重点分析然后生成摘要。

信息提取是自然语言处理（NLP）中与从文本中提取你想要了解的某些内容相关的部分。

零样本学习：没有给它任何标记的训练数据，它也能根据提示词生成正确的答案。

### 信息提取的步骤

1. 确定你想要提取的信息
2. 使用适当的提示词来提取信息
3. 检查提取的信息是否准确
4. 根据需要调整提示词
5. 重复步骤 2-4，直到得到满意的结果

> Redlines: python 库，用于文本高亮和标记。高亮标记文本的特定部分，以便更容易阅读和理解。

## 聊天机器人

有 3 个角色：system assistant user
system: 系统角色，负责设置聊天机器人的行为和规则
assistant: 助手角色，负责与用户进行对话
user: 用户角色，负责与助手进行对话
